1287. Element Appearing More Than 25% In Sorted Array

Easy

Given an integer array sorted in non-decreasing order, there is exactly one integer in the array that occurs more than 25% of the time.

Return that integer.

Example 1:

Input: arr = [1,2,2,6,6,6,6,7,10]
Output: 6

Constraints:

  • 1 <= arr.length <= 10^4
  • 0 <= arr[i] <= 10^5

[Solution]

class Solution {
    public int findSpecialInteger(int[] arr) {
        int n = arr.length;
        for (int i = 0 ; i < n; i += n / 4) {
            int cnt = findElement(arr, arr[i]);
            if (cnt > n/4) return arr[i];
        }
        
        return -1;
    }
    
    int findElement(int[] arr, int target) {
        int l = 0, u = arr.length - 1;
        while (l < u) {
            int mid = l + (u -l) / 2;
            if (arr[mid] < target) l = mid+1;
            else u = mid;
        }
        
        int lower = arr[u] == target ? u : -1;
        
        if (lower == -1) return -1;
        
        u = arr.length - 1;
        while (l < u) {
            int mid = l + (u -l) / 2;
            if (arr[mid] <= target) l = mid+1;
            else u = mid;
        }
        
        int upper = arr[u] == target ? u + 1 : u;
        return upper - lower;
    }
}
